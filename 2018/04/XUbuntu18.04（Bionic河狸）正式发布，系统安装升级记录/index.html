XUbuntu18.04（Bionic河狸）正式发布，系统安装升级记录

Ubuntu 18.04发布时间是（美国）4月26号，彼时的中国已经是27号，就是今天。我花了一天时间折腾升级旧Ubuntu系统到刚发布的最新版。

多年使用linux经验已经使得这个过程驾轻就熟，闲庭信步:)。为了不忘记，将有沉淀价值的部分，简要记录一下。

偶然得知Linux，成为Ubuntu多年用户¶

建议：只使用Ubuntu的LTS版本（长期维护）¶

我的Ubuntu升级之路是10.04 -> 14.04 -> 18.0，中间的LTS版本12.04和16.04因在升级后不满意而没用，回滚恢复降版本。如果升级前没有备份，就没法恢复。系统升级前备份旧系统的准备很重要，让你可以安心折腾走得更远。这次也是花了很长时间全备份的。如果失败需要恢复系统可参考关于linux系统备份与恢复的随笔

期待已久的XUbuntu18.04（Bionic河狸）正式发布¶

新系统意味着更好的驱动，更好的软件版本，比如ubuntu14.04驱动RTL8723BE这种网卡会偶尔掉线。不幸，我的一台笔记本就是RTL8723BE网卡的。显然大版本升级 14.04 -> 18.04 能巧妙地不折腾而解决很多旧问题（再比如http2支持）。因此，我是带着期待在等待Ubuntu18.04发布的，同时这次升级也没有退路（14.04太老了）。幸好，升级还算满意。已经用上了最新Ubuntu了，并且速度还可以，在换SSD盘后的酷睿I3老机上运行顺畅。

Ubuntu 18.04 桌面截图(XUbutnu)¶

Ubutnu18.04安装升级过程 （Xfce版本）¶

1.0 选择并下载Ubutnu版本(XUbuntu)¶

旧系统是配搭Xfce桌面的XUbuntu14.04，Xfce同时也是Linus本人选择Linus Torvalds 弃 GNOME 3 用 Xfce。XUbuntu里面的好软件有很多，比如文件管理器Thunar，截图软件xfce4-screenshooter，网速提示器xfce4-netload-plugin，CPU内存提示器xfce4-systemload-plugin等等。同时Ubuntu18的Xfce比原版ISO文件轻量很多。

2.0 升级前先备份¶

2.1 备份已安装过的deb包名称列表¶

( zcat /var/log/apt/history.log*.gz | grep Commandline: ;  zcat /var/log/apt/history.log*.gz | grep Commandline: ) > ~/deb.list.txt

2.2 备份已安装过的python包名称列表（无则忽略，ruby，perl等备份操作同理）¶

( pip2 list ; pip3 list ) > ~/pip.list.txt

2.3 /etc,/var一些关键目录备份, root权限执行（/etc是配置文件仓库，/var包含各用户的定时任务)¶

cd /; sudo tar czf ~/etc.tar.gz etc var/spool/cron/crontabs

2.4 繁重但强烈建议的全盘备份¶

备份方法多种：rsync同步到安全盘备份 或 tar打包放到安全目录下 或 dd复制分区为文件保存到安全目录下。

3.0 分区、格式化、安装¶

3.1 方式一：从新硬盘或空白分区安装ubuntu18.04¶

linux分区不是越多越好的，推荐 / + /home 模式， 实用而不折腾， 一个分区代表系统，一个分区代表用户数据空间。这样的好处就是为后续升级带来方便。

根 / 分区给15G已经完全足够，绰绰有余了。 把剩余空间都给/home，因为/分区使用量不会明显增加，/home分区放的东西却会成倍的增加。

3.2 方式二：从旧linux系统升级¶

告诫：不要尝试从命令sudo apt-get dist-upgrade升级，结局一般都是不能自动处理错误而中止升级，系统可能因此寿终正寝，只留下无辜的你。这和不要尝试用系统升级方法装windows是一样的。

参考升级方法：

  1. 之前已有linux分区是 / + /home 模式，先把 “/” 所在分区格式化，然后将整个ubuntu18.04安装 “/” 所在区。
  2. 系统安装后重启进入ubuntu18.04系统，原 “/home” 分区不会挂载。 总过程已经完成一半了。
  3. 再挂载并将原 “/home” 分区清理瘦身一下，删除家目录中.thumbnails, .cache这些可能冲突或不大重要的用户数据。
  4. 将新/home目录内容rsync同步到旧 “/home” 分区所在目录里。
  5. 修改/etc/fstab，增加1行让系统重启后挂载原 “/home” 盘。确认无误后重启就完成全部过程。

这样的好处是家目录中的图片、视频、工作文件、工作目录等用户资料依然在那里，变了的只是系统部分，优雅完成升级。

4.0 安装Ubuntu18.04系统之后的优化¶

4.1 Ubuntu18.04修改源¶

原来的url是cn.archive.ubuntu.com， 速度下载包时有3MB以上。其实不换也是可以的。

如果要换， 以阿里举例， 修改/etc/apt/sources.list文件内容即可（修改前备份）， 改其他第三方的同理该域名即可。

deb http://mirrors.aliyun.com/ubuntu/ bionic main restricted

deb http://mirrors.aliyun.com/ubuntu/ bionic-updates main restricted

deb http://mirrors.aliyun.com/ubuntu/ bionic universe
deb http://mirrors.aliyun.com/ubuntu/ bionic-updates universe

deb http://mirrors.aliyun.com/ubuntu/ bionic multiverse
deb http://mirrors.aliyun.com/ubuntu/ bionic-updates multiverse

deb http://mirrors.aliyun.com/ubuntu/ bionic-backports main restricted universe multiverse

deb http://mirrors.aliyun.com/ubuntu bionic-security main restricted
deb http://mirrors.aliyun.com/ubuntu bionic-security universe
deb http://mirrors.aliyun.com/ubuntu bionic-security multiverse

4.2 Ubuntu18.04调整用户界面、桌面工具栏、安装字体、重新安装历史包等¶

4.2.1 调整桌面工具栏到下面，下重上轻的风格看起来比较稳重。也是xp时代传下的习惯。¶

XUbutnu18.04截图(Ubuntu)，这个版本别名”Bionic Beaver”仿生河狸¶

4.2.2 在桌面地板面板工具栏上，添加自定义显示比如网速，CPU，内存等。¶

添加网速提示器xfce4-netload-plugin，CPU内存提示器xfce4-systemload-plugin后的工具栏随时可获知负载网速这些计算机情况，用户体验比windows更赞！

4.2.3 安装谷歌拼音，文泉驿字体，调整系统字体大小等¶

Ubuntu18.04默认安装fcitx-sunpinyin也好用，用起来和googlepinyin差不多。所以……我还是选大款的好了。

sudo apt-get install fcitx-googlepinyin fonts-wqy*

4.3 实现QQ截图功能，快捷键Ctrl+Alt+A框选截屏。(linux下用xfce-screenshooter变身QQ截图神器)¶

不需安装额外软件xfce-screenshooter是XUbuntu自带的，只需操作如下步骤： 打开设置-> 键盘 -> 应用程序快捷键 -> 添加。 然后命令框输入xfce4-screenshooter –region确认，再按住Ctrl+Alt+A，确认添加即可。

xfce4-screenshooter这软件很赞，不需安装QQ就能有很好的截图体验。某些时候，linux用户体验可以比windows好，需要使用者本身已经对linux比较上手。

5.0 安装Ubuntu18.04系统之后的问题解决¶

5.1 Ubuntu18.04启动shadowsocks报错解决¶

原因：openssl升级到1.1.0以上版本，导致shadowsocks启动报undefined symbol: EVP_CIPHER_CTX_cleanup错误

解决： 将py文件中所有的EVP_CIPHER_CTX_cleanup改为EVP_CIPHER_CTX_reset

sudo vim /usr/local/lib/python2.7/dist-packages/shadowsocks/crypto/openssl.py  # 按实际路径改

5.2 Ubuntu18.04安装wps（解决报错error while loading shared libraries: libpng12.so.0)¶

第一步下载wps-office_10.1.0.5672~a21_amd64.deb, libpng12-0_1.2.54-1ubuntu1_amd64.deb, wps_symbol_fonts.zip（window字体非必须）文件备用。

第二步运行命令安装：

sudo dpkg -i wps-office_10.1.0.5672~a21_amd64.deb libpng12-0_1.2.54-1ubuntu1_amd64.deb

5.3 底部面板panel出现重复的网络图标解决¶

解决：设置 - 会话与启动 - 程序自启动 - 取消勾选network项。 注销，再登录，重新勾选，注销，再登录。 图标恢复只有一个。

升级过程比较有把握，遇到的问题都是典型缺so文件这些小问题，通过谷歌搜索就能解决。

某些时候，linux用户体验可以比windows好，需要使用者本身已经对linux比较上手。
