id="_1">最长公共子串问题分析<a class="headerlink" href="#_1" title="Permanent link">&para;</a></h3> <p>其实这是一个序贯决策问题，可以用动态规划来求解。我们采用一个二维矩阵来记录中间的结果。这个二维矩阵怎么构造呢？直接举个例子吧：&rdquo;bab&rdquo;和&rdquo;caba&rdquo;(当然我们现在一眼就可以看出来最长公共子串是&rdquo;ba&rdquo;或&rdquo;ab&rdquo;)</p> <p>b　　a&nbsp;b</p> <p>c　　0　　0&nbsp;0</p> <p>a　　0　　1&nbsp;0</p> <p>b　　1　　0&nbsp;1</p> <p>a　　0　　1&nbsp;0</p> <p>我们看矩阵的斜对角线最长的那个就能找出最长公共子串。</p> <p>不过在二维矩阵上找最长的由1组成的斜对角线也是件麻烦费时的事，下面改进：当要在矩阵是填1时让它等于其左上角元素加1。</p> <p>b　　a&nbsp;b</p> <p>c　　0　　0&nbsp;0</p> <p>a　　0　　1&nbsp;0</p> <p>b　　1　　0&nbsp;2</p> <p>a　　0　　2&nbsp;0</p> <p>这样矩阵中的最大元素就是&nbsp;最长公共子串的长度。</p>
